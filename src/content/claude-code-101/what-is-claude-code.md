Claude Code는 여러분의 코드베이스를 이해하고, 파일을 직접 편집하며, 명령어를 실행하고, 기존 개발 도구와 통합되어 일을 더 빠르게 끝낼 수 있도록 돕는 **에이전틱(agentic) 코딩 도구**입니다. 터미널, Visual Studio Code, Claude Desktop 앱, 웹, JetBrains IDE에서 모두 사용할 수 있어요.

## Claude와 Claude Code는 어떻게 다른가요?

Claude.ai를 써본 적이 있다면 "Claude Code는 뭐가 다른가" 궁금할 수 있습니다. Claude.ai와 달리 **Claude Code는 여러분의 파일·터미널·코드베이스 전체에 직접 접근**합니다. 코드를 일일이 복사·붙여넣기 하며 주고받을 필요 없이, Claude Code가 직접 들어가서 작업을 수행해요.

핵심 차별점은 Claude Code가 **AI 에이전트(AI Agent)** 로 동작한다는 점입니다.

## 그럼 에이전트(Agent)란 무엇인가요?

AI 에이전트는 **자신이 놓인 환경과 상호작용하면서 정해진 목표를 달성하기 위해 행동을 수행하는 소프트웨어**입니다. 그 본질은 *대형 언어 모델(LLM)이 실시간으로 루프 안에서 동작*하는 구조죠. AI 에이전트는 도구, 외부 서비스, 심지어 다른 AI 에이전트에게도 접근해 자신의 목표를 향해 나아갈 수 있습니다.

> 즉, 모델이 한 번 답변을 내고 끝나는 것이 아니라, **관찰 → 판단 → 행동 → 다시 관찰**의 사이클을 스스로 반복하며 일을 진행하는 것이 "에이전틱"이라는 용어의 핵심입니다.

## Claude Code가 실제로 할 수 있는 일

실무에서 Claude Code가 어떤 일을 해주는지 정리하면 이렇습니다.

- **코드베이스를 읽고 이해합니다.** 특정 기능의 동작을 설명해 달라고 하거나, 코드 전반에 걸쳐 버그의 흐름을 추적하게 할 수 있어요.
- **프로젝트 전반의 파일을 편집합니다.** 함수 하나를 리팩터링한 뒤, 그 함수를 참조하는 모든 파일까지 같이 업데이트하는 식입니다.
- **터미널 명령을 실행합니다.** 빌드 스크립트를 돌리고, 테스트를 실행하고, 패키지를 설치하고, 그 결과를 보고 다음 행동을 스스로 결정해요.
- **웹을 검색합니다.** 문서나 최신 API 레퍼런스가 필요하면 직접 찾아봅니다.

## Claude Code를 효과적으로 쓰는 세 가지 감각

Claude Code를 잘 활용하려면 다음 세 가지 개념을 머릿속에 두고 쓰는 것이 좋습니다.

**1) 컨텍스트 윈도우(Context window).** Claude의 *작업 기억(working memory)* 이라고 생각하면 됩니다. 꽤 많은 정보를 담을 수 있지만, 한꺼번에 전부는 아닙니다. 그래서 "에이전틱"이라는 특성이 의미가 있는데, Claude는 코드베이스 전체를 통째로 메모리에 올리지 않고 **필요한 답을 찾아 들어가는 전략**을 스스로 선택합니다.

**2) 권한을 묻습니다(It asks for permission).** Claude Code는 기본적으로 명령을 실행하거나 변경을 가하기 전에 여러분에게 확인을 요청합니다. 손수 하나하나 챙기는 스타일이든, 더 자율적으로 맡기는 스타일이든 — **주도권은 항상 여러분에게** 있습니다.

**3) 실수할 수 있습니다(It can make mistakes).** 다른 도구들과 마찬가지로 Claude Code도 완벽하지 않습니다. 의도를 잘못 이해하거나, 버그를 만들거나, 과하게 설계할 때가 있어요. **루프 안에서 함께 보고 있는 것**이 이런 상황을 일찍 잡아내는 가장 좋은 방법입니다.

## 정리

Claude Code는 **에이전틱 코딩 도구**입니다. 코드베이스를 읽고, 파일을 편집하고, 명령을 실행하고, 외부 도구에 연결되어 출시 속도를 끌어올려 줍니다. 지금 바로 **터미널, VS Code, JetBrains, Claude Desktop 앱**에서 사용해 볼 수 있어요.
