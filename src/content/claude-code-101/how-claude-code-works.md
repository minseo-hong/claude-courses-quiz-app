Claude Code는 일반적인 채팅 애플리케이션과 다릅니다. 내부에서 어떻게 동작하는지 감을 잡아 두면 훨씬 더 효과적으로 활용할 수 있어요.

## 에이전틱 루프(Agentic Loop)

Claude Code의 동작은 **에이전틱 루프**라는 개념으로 가장 잘 설명됩니다.

1. 여러분이 Claude Code에 프롬프트를 입력합니다.
2. Claude는 모델과 상호작용하며 필요한 **컨텍스트(context)** 를 모읍니다. 모델은 텍스트를 돌려주거나, Claude Code가 실행할 수 있는 **도구 호출(tool call)** 을 돌려줍니다.
3. **행동을 수행합니다.** 예를 들어 파일을 편집하거나 명령어를 실행하는 식이죠.
4. **결과를 검증합니다.** 그 결과가 여러분이 프롬프트로 요청한 일을 실제로 달성했는지 확인합니다.
5. 달성했다면 Claude는 작업을 마치고 다음 프롬프트를 기다립니다. 그렇지 않으면 다시 루프로 돌아가, 결과가 완성되고 검증될 때까지 시도를 반복합니다.

루프가 도는 동안에도 여러분은 언제든 **컨텍스트를 추가**하거나, **작업을 중단**하거나, 모델의 **방향을 다시 잡아** 목표 쪽으로 끌어당길 수 있습니다.

## 컨텍스트(Context)

Claude에는 **컨텍스트 윈도우(context window)** 가 있습니다. 대화 내용, 파일 내용, 명령어 출력 등을 한 번에 얼마나 저장하고 참조할 수 있는지를 결정하는 용량이죠. 이 한도에 도달하면 Claude Code는 대화를 **자동으로 압축(compact)** 합니다 — 무엇을 제거하거나 요약할지 스스로 결정해, 컨텍스트 윈도우를 다시 사용 가능한 크기로 줄여 놓습니다.

## 도구(Tools)

도구는 에이전트가 일을 하는 데 있어 가장 근본적인 골격입니다. 대부분의 AI 어시스턴트는 단순히 텍스트를 받아 텍스트를 돌려주는 데서 끝납니다. 반면 도구는 Claude Code가 작업을 완수하는 데 한 발짝 더 가까워지기 위해 *언제 코드를 실행해야 하는지* 를 스스로 판단할 수 있게 해 줍니다. 파일을 읽는 도구일 수도 있고, 웹 검색 도구일 수도 있고, 그 밖의 어떤 능력이든 도구가 될 수 있어요. Claude Code는 **의미 기반 이해(semantic understanding)** 를 통해 언제 어떤 도구를 호출해야 하는지, 그리고 그 출력을 어떻게 활용해야 하는지 결정합니다.

## 권한(Permissions)

Claude Code에는 여러 권한 모드가 있습니다.

- **기본 동작(Default behavior):** 파일을 편집하거나 셸 명령어를 실행하기 전에 명시적으로 허가를 요청합니다.
- **자동 수락(Auto-accept):** 파일은 묻지 않고 편집하지만, 명령어 실행은 여전히 승인을 받습니다.
- **계획 모드(Plan mode):** 읽기 전용 도구만 사용해, 실제 작업에 들어가기 전에 행동 계획을 먼저 정리합니다.

![Claude Code가 bash 명령어를 실행하기 전에 사용자에게 권한을 묻는 모습](https://everpath-course-content.s3-accelerate.amazonaws.com/instructor%2F8lsy243ftffjjy1cx9lm3o2bw%2Fpublic%2F1775686376%2Fvideo2ask.1775686376586.jpg)

이 모든 동작은 설정 파일에서 구성할 수 있습니다. 다만 권한을 건너뛸 때는 신중하세요 — Claude Code에 명령어를 자유롭게 실행할 권한을 열어 주면, 실수가 일어나기 전에 그것을 미리 잡아내기가 더 어려워질 수 있습니다.

## 정리

Claude Code는 여러 에이전틱 개념을 한데 묶어 놓은 도구입니다 — **에이전틱 루프**, **관리되는 컨텍스트 윈도우**, **도구**, 그리고 **설정 가능한 권한** — 이 모든 것이 여러분의 터미널 안에서 동작합니다. 코드베이스를 읽고, 행동을 수행하고, 자기 자신의 작업을 검증하죠. 바로 이 점이 Claude Code를 단순한 채팅 창과 근본적으로 다르게 만듭니다.
