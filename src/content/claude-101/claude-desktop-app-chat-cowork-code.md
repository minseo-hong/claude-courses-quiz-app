> **예상 소요 시간 6분**

## 학습 목표

이 레슨을 마치면 다음을 할 수 있게 됩니다.

- 데스크톱 앱이 제공하는 세 가지 모드 — **Chat**, **Cowork**, **Code** — 가 각각 어떤 작업에 어울리는지 구분할 수 있다
- 각 모드의 핵심 기능(Quick entry·예약 작업·로컬/원격 개발 등)이 무엇이고 왜 그 모드에 들어가 있는지 설명할 수 있다
- 작업 성격에 따라 어떤 모드를 선택해야 가장 빠르게 끝낼 수 있는지 판단할 수 있다

## 세 모드 한눈에 보기

데스크톱 앱은 같은 Claude를 **세 가지 다른 작업 입구**로 보여 줍니다.

- **Chat**은 Claude.ai에서 쓰던 그 Claude에, 데스크톱에서만 가능한 빠른 호출·스크린샷·받아쓰기·로컬 커넥터가 더해진 모드입니다.
- **Cowork**는 *agentic* 작업용입니다. 목표를 정해 주고 도구·자료를 연결해 두면 Claude가 스스로 단계를 밟아 결과물을 만들어 옵니다. 깊이 있는 리서치, 다중 출처 분석, 완성도 높은 산출물에 적합합니다.
- **Code**는 소프트웨어를 만드는 모드입니다. 코드 작성·테스트·실행·배포까지 한 자리에서 다룹니다.

> Cowork와 Code는 같은 엔진(Claude Code) 위에서 돌아갑니다. 둘 다 로컬에서 독립적으로 작동하며, 서브에이전트를 띄우거나 장시간 작업을 이어 갈 수 있어 큰 작업을 모델이 스스로 진행할 수 있습니다.

## Chat — 묻고 다듬는 빠른 대화

![Chat 모드 화면](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/69a8c30634a6c3e545269791_69a8a456caa28723041a6eea_699e1b746d83c9fb55a56d31_698be3352261cf694fdae308_698bdd295c5f5d680c01e63c_Screenshot%25252525202026-02-09%2525252520at%25252525204.38.22%25252525E2%2525252580%25252525AFPM.png)

질문하기, 브레인스토밍, 초안 다듬기, 문제를 함께 풀어 가기 — 짧은 호흡으로 주고받는 작업에 가장 잘 맞습니다. Claude.ai와 동일한 사용 경험에 데스크톱에서만 쓸 수 있는 다음 기능이 더해집니다.

- **Quick entry.** macOS에서 **Option 키를 더블탭**하면 작업 중인 화면 위로 컴팩트한 Claude 창이 떠 작업 흐름을 끊지 않고 질문할 수 있습니다.
- **스크린샷·창 공유.** 보고 있는 화면을 그대로 캡처하거나 특정 창을 공유하면, 화면을 말로 묘사할 필요 없이 Claude가 정확히 같은 것을 보고 답합니다. *(macOS)*
- **받아쓰기(Dictation).** 키보드 대신 말로 생각을 풀어낼 수 있어, 출퇴근길이나 회의 직전처럼 손이 안 닿는 상황에 유용합니다. *(macOS)*
- **데스크톱 커넥터.** 로컬 도구·서비스를 커넥터로 연결해 같은 컴퓨터의 다른 앱 데이터를 함께 다룰 수 있습니다.

**이럴 때 잘 어울립니다.**

- 낯선 대시보드를 보다가 "이 지표 무슨 뜻이지?" 싶을 때 — Option 더블탭 → 영역 캡처 → 그대로 질문하면 화면을 가린 채 답이 옵니다.
- 회의 사이에 짧게 발표 구조를 정리하고 싶을 때 — Quick entry에서 음성으로 풀어 놓으면 Claude가 그 자리에서 개요로 정리해 줍니다.
- 메모 앱에 흩어져 있던 아이디어를 끌어모을 때 — 메모 커넥터를 켜고 "최근에 적어 둔 X 관련 메모를 모아 정리해 줘"라고 부탁하면 됩니다.

## Cowork — 자료를 가로질러 결과물을 만드는 모드

![Cowork 모드 화면](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/69a8c30634a6c3e54526979c_69a8a456caa28723041a6ee7_699e2c4828218e71872531df_699e289b7bffb76b5ebedc79_Screenshot%252525202026-02-24%25252520at%252525202.39.06%252525E2%25252580%252525AFPM.png)

여러 출처에서 정보를 끌어모아 의미를 만들어 내고, *완성된 산출물*까지 만들어야 할 때 빛나는 모드입니다. Cowork는 작업을 동시에 여러 갈래로 펼쳐 진행할 수 있어, 더 많은 소스를 다루고 끝까지 끌고 갈 체력이 있습니다 — 깊이 있는 리서치 브리프, 출처를 가로지르는 재무 분석, 처음부터 끝까지 검토하는 계약서, 흩어진 자료를 모아 만든 슬라이드 등.

작업을 시작하기 전, Cowork는 보통 짧은 질문 몇 개로 범위·형식·제약을 확인하고 사이드바에 검토 가능한 **계획**을 만듭니다. 작업이 진행되는 동안 어떤 출처를 보고 있는지, 어떤 파일이 만들어지고 있는지, 계획의 어느 단계에 있는지가 그대로 보입니다. 여러 작업을 동시에 돌리면서 사이드바에서 자유롭게 오갈 수도 있습니다.

- **폴더 접근.** 폴더 하나를 가리키면 그 안의 자료를 읽어 적합한 것을 추리고 결과물을 같은 자리에 저장합니다. 파일 업로드·내용 붙여넣기·커넥터 연결도 함께 쓸 수 있습니다.
- **예약 작업(Scheduled tasks).** 매일 아침 인박스 정리, 매주 출시 항목 요약, 매일 한 줄 브리핑처럼 반복 업무를 정해진 시간에 돌립니다. 컴퓨터·앱이 꺼져 있었으면 다시 돌아왔을 때 *catch up* 합니다.
- **서브에이전트.** 복잡한 요청을 여러 하위 작업으로 쪼개 각자 별도 컨텍스트에서 동시에 진행하고, 그 결과를 합쳐 하나의 산출물로 돌려줍니다.
- **Dispatch.** 모바일 앱에서 데스크톱의 Cowork 대화를 이어 갑니다. 외출 중에도 컴퓨터 안의 파일·커넥터·플러그인·다른 앱까지 그대로 활용해 작업을 위임할 수 있고, 데스크톱이 깨어 있고 앱이 열려 있어야 동작합니다.
- **프로젝트.** 관련된 작업을 전용 워크스페이스로 묶어 파일·맥락·지침·기억을 한 곳에서 관리합니다. Claude.ai의 Projects와 비슷하지만 Cowork 작업 흐름에 맞춰 로컬에 자리 잡습니다.
- **브라우저 사용.** Claude in Chrome을 통해 사이트를 탐색하고 페이지와 상호작용해 자료를 가져옵니다 — API가 없는 페이지 데이터 수집·경쟁사 가격 비교 같은 일이 여기 들어옵니다.
- **컴퓨터 사용(Computer use).** 커넥터·플러그인이 없을 때는 사람처럼 직접 클릭하고 입력해 작업합니다. 우선순위는 **커넥터 → 브라우저 → 화면 직접 조작** 순으로 가장 빠르고 안정적인 경로를 먼저 시도합니다. 앱 접근 시 권한 확인이 뜨고, 차단 목록도 지정할 수 있습니다. *(Pro·Max에서 research preview, macOS 우선 — Windows 추후)*
- **플러그인.** 실시간 금융 데이터, 사내 지식 베이스 검색, 특정 컴플라이언스 환경 동작처럼 *그 작업에만 필요한* 능력을 추가로 붙여 줍니다.
- **보호된 환경.** Cowork는 컴퓨터 안의 *contained workspace*에서 동작합니다. 공유한 폴더 안에서만 읽고·만들고·고치며, 그 바깥은 건드리지 않습니다.

**이럴 때 잘 어울립니다.**

- 회의록·Slack·이메일을 데이터베이스처럼 질의해 *지난 분기 가격 결정 → Q3 슬라이드 반영* 같은 작업을 한 줄로 끝내고 싶을 때.
- 새로운 시장이나 경쟁사를 조사할 때 — 브라우저 탭을 직접 열지 않고도 사이트들을 돌며 보고서·가격을 모아 출처가 붙은 정리본을 받아 보고 싶을 때.
- 50개 넘는 계약서·재무 보고서·회의록 폴더를 통째로 넘기며 핵심 메모만 받아 보고 싶을 때 — 50개를 5개처럼 검토할 수 있습니다.
- 매일 아침 같은 잡무를 반복하고 있다면, 한 번 예약 작업으로 만들어 두고 Claude가 매일 처리하게 둡니다.

> Cowork는 Pro·Max·Team·Enterprise에서 사용할 수 있고, 새로운 기능이 꾸준히 추가되고 있습니다.

## Code — 소프트웨어를 짓는 자리

![Code 모드 화면](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/69a8c30634a6c3e545269794_69a8a455caa28723041a6ee1_699e1b736d83c9fb55a56d2a_698be3342261cf694fdae301_698bdd78a6031117e6bb41d6_Screenshot%25252525202026-02-09%2525252520at%252525252012.52.07%25252525E2%2525252580%25252525AFPM.png)

데스크톱 앱 안에서 곧바로 **Claude Code**의 능력을 꺼내 쓰는 탭입니다. Code 탭은 코드베이스 안에서 직접 동작합니다 — 읽고, 수정하고, 명령을 돌립니다. **시각 diff**가 변경 사항을 보여 주고, 내장 터미널이 실행 로그를 띄우며, **git**이 모든 버전을 추적해 언제든 되돌릴 수 있습니다.

Cowork가 *공유한 폴더 안*으로 자신을 가두는 contained workspace였다면, Code는 프로젝트 자체에서 동작하며 **파일 시스템·터미널·개발 도구 전체에 접근**합니다.

작업이 일어나는 자리는 두 가지 중 골라 둘 수 있습니다.

- **Local.** 컴퓨터의 폴더를 지정하면 Claude가 그 파일을 직접 다룹니다. 로컬에서 도는 만큼 개발 서버를 띄워 브라우저로 미리보기를 띄우는 식의 흐름이 자연스럽습니다.
- **Remote.** GitHub 저장소를 연결하면 클라우드 환경에서 작업이 진행됩니다. 앱을 닫아도 세션이 살아 있어, 큰 리팩터를 시작해 두고 한참 뒤에 들여다보는 식의 작업이 가능합니다 — 큰 코드베이스나 로컬을 비워 두고 싶을 때 좋습니다.

자율성은 세 가지 상호작용 모드로 조절합니다.

- **Ask.** 모든 변경을 Claude가 *제안*하고 사용자가 diff를 검토한 뒤 승인해야 적용됩니다.
- **Code.** 파일 변경은 자동으로 적용하되, 터미널 명령은 실행 전에 한 번 더 확인합니다.
- **Plan.** 무언가를 건드리기 전에 전체 접근 방식을 먼저 보여 줍니다. 별도 *플랜 뷰어*에서 전략을 검토하고, 작업 진행 중에도 다시 들여다볼 수 있습니다.

여러 세션을 동시에 띄울 수 있고, 사이드바에서 상태(Active/Archived)·환경(Local/Cloud) 기준으로 필터링할 수 있습니다.

> Code 탭은 Pro·Max·Team·Enterprise에서 사용할 수 있습니다.
